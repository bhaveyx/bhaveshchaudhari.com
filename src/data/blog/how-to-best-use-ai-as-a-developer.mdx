## Introduction

Web performance is crucial for user experience, SEO rankings, and business success. In this comprehensive guide, we'll explore modern performance optimization techniques that can dramatically improve your website's speed and user experience.

<Callout type="info" title="Performance Impact">
Studies show that a 1-second delay in page load time can result in a 7% reduction in conversions and an 11% decrease in page views.
</Callout>

## Understanding Core Web Vitals

Google's Core Web Vitals are essential metrics for measuring user experience:

### Largest Contentful Paint (LCP)

LCP measures loading performance. To provide a good user experience, LCP should occur within **2.5 seconds** of when the page first starts loading.

<InfoCard title="LCP Optimization Strategies">
- Optimize server response times
- Use efficient image formats (WebP, AVIF)
- Implement proper caching strategies
- Minimize render-blocking resources
</InfoCard>

### First Input Delay (FID)

FID measures interactivity. To provide a good user experience, pages should have an FID of **100 milliseconds** or less.

### Cumulative Layout Shift (CLS)

CLS measures visual stability. To provide a good user experience, pages should maintain a CLS of **0.1** or less.

## Performance Optimization Techniques

### Image Optimization

Images often account for the majority of downloaded bytes on a web page. Here's how to optimize them:

<CodeBlock title="Modern Image Optimization" language="html">
{`<!-- Use modern formats with fallbacks -->
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description" loading="lazy" decoding="async">
</picture>

<!-- Responsive images -->
<img 
  srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
  sizes="(max-width: 480px) 100vw, (max-width: 800px) 50vw, 25vw"
  src="medium.jpg"
  alt="Description"
  loading="lazy"
>`}
</CodeBlock>

### Critical Resource Optimization

<CodeBlock title="Resource Hints and Preloading" language="html">
{`<!-- Preload critical resources -->
<link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/css/critical.css" as="style">

<!-- DNS prefetch for external domains -->
<link rel="dns-prefetch" href="//fonts.googleapis.com">
<link rel="dns-prefetch" href="//analytics.google.com">

<!-- Preconnect to critical third-party origins -->
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>`}
</CodeBlock>

### JavaScript Optimization

<CodeBlock title="Efficient JavaScript Loading" language="javascript">
{`// Code splitting with dynamic imports
const loadComponent = async () => {
  const { default: Component } = await import('./HeavyComponent')
  return Component
}

// Intersection Observer for lazy loading
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      loadComponent().then(Component => {
        // Render component
      })
      observer.unobserve(entry.target)
    }
  })
})

// Service Worker for caching
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js')
    .then(registration => console.log('SW registered'))
    .catch(error => console.log('SW registration failed'))
}`}
</CodeBlock>

<QuickTip>
Use the Coverage tab in Chrome DevTools to identify unused JavaScript and CSS that can be removed or lazy-loaded.
</QuickTip>

## Advanced Performance Patterns

### Resource Prioritization

<CodeBlock title="Resource Priority Hints" language="html">
{`<!-- High priority for above-the-fold images -->
<img src="hero.jpg" fetchpriority="high" alt="Hero image">

<!-- Low priority for below-the-fold content -->
<img src="footer-logo.jpg" fetchpriority="low" alt="Footer logo">

<!-- Auto priority (default) -->
<img src="content.jpg" fetchpriority="auto" alt="Content image">`}
</CodeBlock>

### Performance Budgets

Set performance budgets to maintain fast loading times:

<InfoCard title="Recommended Performance Budgets">
- **Total page weight**: < 1.5MB
- **JavaScript bundle**: < 300KB
- **Images**: < 800KB
- **Fonts**: < 100KB
- **Time to Interactive**: < 3.5s on 3G

### Monitoring and Measurement

<CodeBlock title="Performance Monitoring" language="javascript">
{`// Web Vitals measurement
import { getCLS, getFID, getFCP, getLCP, getTTFB } from 'web-vitals'

function sendToAnalytics(metric) {
  // Send to your analytics service
  gtag('event', metric.name, {
    event_category: 'Web Vitals',
    value: Math.round(metric.name === 'CLS' ? metric.value * 1000 : metric.value),
    event_label: metric.id,
    non_interaction: true,
  })
}

getCLS(sendToAnalytics)
getFID(sendToAnalytics)
getFCP(sendToAnalytics)
getLCP(sendToAnalytics)
getTTFB(sendToAnalytics)

// Performance Observer for custom metrics
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.entryType === 'navigation') {
      console.log('Navigation timing:', entry)
    }
  }
})

observer.observe({ entryTypes: ['navigation', 'resource', 'paint'] })`}

## Framework-Specific Optimizations

### Next.js Performance

<CodeBlock title="Next.js Optimization" language="javascript">
{`// next.config.js
module.exports = {
  // Enable experimental features
  experimental: {
    optimizeCss: true,
    optimizeImages: true,
  },
  
  // Image optimization
  images: {
    formats: ['image/avif', 'image/webp'],
    deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840],
  },
  
  // Compression
  compress: true,
  
  // Bundle analyzer
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.resolve.fallback.fs = false
    }
    return config
  },
}`}
</CodeBlock>

### React Performance

<CodeBlock title="React Performance Patterns" language="javascript">
{`import { memo, useMemo, useCallback, lazy, Suspense } from 'react'

// Memoized component
const ExpensiveComponent = memo(({ data, onUpdate }) => {
  const processedData = useMemo(() => {
    return data.map(item => ({
      ...item,
      processed: true
    }))
  }, [data])

  const handleUpdate = useCallback((id) => {
    onUpdate(id)
  }, [onUpdate])

  return (
    <div>
      {processedData.map(item => (
        <Item key={item.id} data={item} onUpdate={handleUpdate} />
      ))}
    </div>
  )
})

// Lazy loading
const LazyComponent = lazy(() => import('./LazyComponent'))

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </Suspense>
  )
}`}
</CodeBlock>
